## Abstract

LynxFlow v2 replaces the two-layer SPL2+sugar language of RFC-001 with a single clean
language. It is a **clean break**: RFC-001's principle P4 (full SPL2 backward
compatibility) is dropped. The language keeps the name **LynxFlow**, promoted from sugar
layer to *the* language. SPL2 ceases to exist as a runtime layer after the migration
window (PLAN.md §18.2).

The design rests on four pillars:

1. **One expression grammar** with standard precedence — no SEARCH/WHERE/EVAL split.
2. **Typed values** — native `array`, `object`, `duration`; explicit casts; no silent
   cross-type coercion.
3. **Null ≠ missing ≠ parse-error**, with parse failures observable as filterable
   `_error` data, never silent.
4. **Index-honest text search** — `has` (term index, fast) vs `contains` (substring
   scan) vs `matches` (regex); the cost tier is in the operator name.

This document is the **frozen normative specification** for Phases 0–10 of the RFC-002
refactoring plan. Where the planning document (`PLAN.md`) and the synthesis spec
disagreed, PLAN.md governs; the conflict resolutions and all open-question decisions are
recorded in §14 (Decision log). §15 maps every RFC-001 capability to its v2 equivalent
or an explicit kill decision.

## 1. Design principles

1. **Pipeline-first, source-first.** `from <source>[range] | stage | stage`. Every
   stage is a relation→relation transform; every query prefix up to any `|` is
   independently executable.
2. **One expression grammar.** A single precedence table used identically in filters,
   computed fields, aggregate arguments, and parse conditions.
3. **Raw is the source of truth; parsing is explicit and staged.** `_raw` is
   byte-preserved and immutable; structure is created by visible `parse` stages (or
   ingest-time acceleration that is semantically equivalent to them).
4. **Typed values, explicit casts, no silent coercion.** Cross-type comparison is a
   plan-time error with a suggested cast, never a silent null or hybrid
   lexical/numeric guess.
5. **Null ≠ missing ≠ parse-error, and failures are data.** Parse failures materialize
   as filterable `_error`/`_error_detail` columns. `exists()`, `is_null()`,
   `is_missing()` are distinct predicates.
6. **Index-honest operators.** The operator name tells you the cost tier. The
   tokenizer contract (§6.1) is a documented language semantic.
7. **Terse at 2AM, rigorous underneath.** Bare terms and `field=value` pairs in the
   search position are first-class sugar with one visible desugaring.
8. **Small orthogonal core + visible-desugar sugar.** ~18 core operators; every sugar
   form has exactly one mechanical expansion shown by `--show-rewritten`/EXPLAIN.
9. **Errors are UX.** Every AST node carries a span; the parser recovers at `|`
   boundaries to report multiple errors; every "not found" gets did-you-mean from the
   live field/function catalogs; semantic type errors fire at plan time.
10. **One source of truth for the grammar.** Operators, functions, and keywords are
    defined in a single registry (`pkg/lynxflow/registry`) that generates the parser
    tables, autocomplete catalogs, EBNF, docs stubs, and the LLM cookbook. Served at
    `/api/v1/catalog`.

## 2. Lexical structure

### 2.1 Encoding, whitespace, comments

- Source text is UTF-8. Whitespace separates tokens and is otherwise insignificant
  outside string literals. Line breaks are permitted between tokens.
- Comments: `// line comment` and nestable `/* block comment */`.
- RFC-001's `--` line comments are gone (`--` is not a token).

### 2.2 Identifiers and field references

- Bare identifier: `[A-Za-z_][A-Za-z0-9_]*`. **Field names are case-sensitive.**
- Dotted path `user.id` is **object access** after parsing (see §4.4 for the
  flat-column resolution rule).
- Any other name is quoted with **backticks**: `` `field-with-dash` ``,
  `` `weird field` ``. Backticks are the *single* identifier-quoting rule; backticked
  dotted names refer to flat columns literally (`` `a.b` `` never means object access).
- Double quotes are **always strings**, never identifiers. Single quotes are not a
  token (parse error with a fix-it pointing at `"` or backticks).
- Keywords are case-insensitive; canonical form is lowercase. Keywords may be used as
  field names only when backticked.

### 2.3 Literals

| Kind | Syntax |
|---|---|
| string | `"text"` with escapes `\" \\ \n \t \r \u{NNNN}`; raw string `r"..."` (no escapes; `"` cannot appear inside) |
| number | `42`, `3.14`, `1e-6`, `0x2a` — int iff no `.`/exponent; hex is int |
| bool / null | `true`, `false`, `null` |
| timestamp | RFC3339 `2026-06-10T08:00:00Z`; via `now()`, `now() - 1h` |
| duration | `30s`, `5m`, `1h`, `7d`, `1.5h`, `100ms` — first-class type |
| array | `[1, 2, "x"]` — ordered, heterogeneous allowed |
| object | `{service: "api", retry: true}` — keys are identifiers or strings; values are expressions |

- Duration units: `ns`, `us`, `ms`, `s`, `m`, `h`, `d`, `w`. No month/year units
  (ambiguous); use absolute timestamps.
- Regex patterns are written as **raw strings** after regex-taking functions:
  `matches(_raw, r"\d+ users?")`. There are no slash-delimited regex literals. Case
  insensitivity via inline `(?i)`. Default engine is linear-time (RE2 class); PCRE2 is
  an explicit opt-in carried over from RFC-001.
- F-strings are removed (§15); use `printf` or string concatenation of strings.

### 2.4 Time ranges and snapping (on `from`)

```
from app[-1h]            // last hour
from app[-7d..-1d]       // between 7 and 1 days ago
from app[@d]             // since start of today (snap)
from app[-1h][@h]        // last hour, snapped to hour boundary
from app[2026-06-01T00:00:00Z..2026-06-02T00:00:00Z]
```

In expressions: `_time > now() - 15m`, `_time between <a> and <b>`. Time-bucket
grouping is the `bin(_time, 5m)` function in a `stats ... by` list.

### 2.5 Source globs

`from logs*,!logs-debug*[-7d..-1d]` — glob includes, `!`-prefixed excludes, `*` for
all sources, `$name` for CTE references. Glob syntax (`*`, `?`, `[a-z]`, `{a,b}`)
carries over from RFC-001 §3.7. Source names needing quoting use backticks.

## 3. Query shape

```
[let $name = <pipeline> ;]*
from <source-list>[<time-range>] [<search-sugar-terms>]
| <stage>
| <stage> ...
```

- **CTEs:** `let $errs = from app[-1h] | where level == "ERROR";` — bound pipelines
  referenced as `from $errs`. The `let` keyword is required (RFC-001's bare
  `$x = ...;` form is gone). `let` exists *only* at query level; there is no `| let`
  stage (use `extend`).
- **Free-hand queries** keep the RFC-001 P8 contract: `lynxdb query 'timeout'` →
  `from <default> | where has(_raw, "timeout")`, with the chosen scope visible in
  `meta.rewrites`.
- A query starting with a pipeline stage (`| stats count()`) gets an implicit
  `from <default>` with a visible rewrite.

### 3.1 Search sugar (the 2AM path)

Immediately after the source (and only there), a sequence of search terms is accepted
and desugared with one visible rewrite:

| Sugar term | Desugars to |
|---|---|
| `timeout` (bare word) | `has(_raw, "timeout")` — case-insensitive, term-index-backed |
| `"connection reset"` (quoted phrase) | `contains(_raw, "connection reset")` — case-insensitive substring; planner extracts terms for index candidates |
| `status=500` | `status == 500` |
| `status>=500`, `<`, `<=`, `>`, `!=` | `status >= 500` etc. |
| `host=web-*` (glob value) | `glob(host, "web-*")` |
| `level in ("a","b")` | `level in ["a", "b"]` |
| juxtaposition | `and` |
| `or`, `not`, parentheses | as written, **standard precedence** (`not` > `and` > `or`) |

Multiple terms: `from nginx[-1h] timeout status>=500` →
`from nginx[-1h] | where has(_raw, "timeout") and status >= 500`.

There is **no Splunk SEARCH precedence**: `a or b c` parses as `a or (b and c)`.
Key:value sugar values follow expression typing rules — `status=500` compares as int
against an int column; against a string column it is a plan-time error with a fix-it
(unknown-type columns compare leniently at runtime, §5.4).

## 4. Expression grammar (the only one)

### 4.1 Precedence

Loosest → tightest:

1. `or`
2. `and`
3. `not` (unary)
4. comparison: `==`, `!=`, `<`, `<=`, `>`, `>=`, `in`, `between … and …`
5. `??` (coalesce)
6. additive: `+`, `-`
7. multiplicative: `*`, `/`, `%`
8. unary minus
9. postfix: call `f(args)`, member `.field`, safe member `?.field`, index `[expr]`
10. primary: literal, identifier, `(expr)`, lambda

Comparisons are non-associative: `a < b < c` is a parse error with a fix-it
(`a < b and b < c`).

### 4.2 EBNF (normative skeleton)

```ebnf
expr        ::= or_expr
or_expr     ::= and_expr ('or' and_expr)*
and_expr    ::= not_expr ('and' not_expr)*
not_expr    ::= 'not' not_expr | cmp_expr
cmp_expr    ::= coal_expr (cmp_op coal_expr
              | 'in' coal_expr
              | 'between' coal_expr 'and' coal_expr)?
cmp_op      ::= '==' | '!=' | '<' | '<=' | '>' | '>='
coal_expr   ::= add_expr ('??' add_expr)*
add_expr    ::= mul_expr (('+' | '-') mul_expr)*
mul_expr    ::= unary_expr (('*' | '/' | '%') unary_expr)*
unary_expr  ::= '-' unary_expr | postfix
postfix     ::= primary (call_args | '.' ident | '?.' ident | '[' expr ']')*
call_args   ::= '(' [arg (',' arg)*] ')'
arg         ::= expr | 'where' expr            // 'where' only in aggregate calls
primary     ::= literal | ident | backtick_ident | '(' expr ')' | lambda
              | array_lit | object_lit
lambda      ::= ident '->' expr
array_lit   ::= '[' [expr (',' expr)*] ']'
object_lit  ::= '{' [obj_entry (',' obj_entry)*] '}'
obj_entry   ::= (ident | string) ':' expr
```

### 4.3 `==` compares; `=` binds

`=` appears **only** in assignment positions: `extend x = …`, stage options
(`limit=10`), and search-position key:value sugar (`status=500`, visible rewrite to
`==`). Using `=` where `==` is meant is a parse error with a fix-it; using `==` in an
assignment position likewise.

### 4.4 Field resolution for dotted paths

`a.b` in an expression resolves, in order:

1. a flat column literally named `a.b` in the current stream schema or field catalog;
2. otherwise, object access `.b` on column `a`.

Backticks force interpretation 1 (`` `a.b` ``); parenthesized access forms force
interpretation 2 (`(a).b`, `a["b"]`). The semantic analyzer warns when both
interpretations are live.

There is **no implicit `_raw` JSON fallback** for unknown dotted fields (RFC-001's VM
behavior). Accessing a field no stage produced is `missing`; a plan-time
unknown-field diagnostic with did-you-mean fires when the catalog proves it never
exists. Use `parse json` to extract structure.

### 4.5 Lambdas

`tags` array functions take lambdas: `any(tags, t -> t.name == "vip")`. A lambda is
valid only as a direct argument to a registered higher-order function; the parameter
shadows any column of the same name inside the body.

## 5. Types and value semantics

### 5.1 Types

`string`, `int` (64-bit signed), `float` (IEEE754 double), `bool`,
`timestamp` (nanosecond UTC instant), `duration` (nanosecond span),
`array`, `object`, `null`.

### 5.2 Null vs missing vs parse-error

| Scenario | Representation | `exists()` | `is_null()` | `is_missing()` |
|---|---|---|---|---|
| field never extracted | missing (absent from row) | false | false | true |
| field present, value null (e.g. JSON `null`) | null value | true | true | false |
| parse stage failed for the row | fields best-effort/missing **plus** `_error`/`_error_detail` populated | per-field | per-field | per-field |

Behavior (applies to both null and missing unless noted):

| Operation | Behavior |
|---|---|
| comparison with null/missing | → null (row fails `where`) |
| arithmetic / string ops with null | → null |
| `and`/`or` | three-valued: `null or true` → true; `null and false` → false; else null |
| aggregation inputs | nulls skipped (`count(x)` counts non-null; `count()` counts rows); all-null group → null |
| group keys | null and missing **each form their own visible group**; a lint fires when the null/missing groups exceed a threshold share of rows |
| sort | nulls last ascending, first descending (deterministic); missing sorts with null |
| `??` | `a ?? b` yields `b` when `a` is null *or* missing |

**There is no error value type in the VM.** Parse errors are data columns
(`_error: string`, `_error_detail: object`) emitted by parse stages (§7.4).

### 5.3 Output rendering

Missing and null both render as `null` in JSON output (the distinction is queryable
via `is_missing`/`is_null`, and visible in grouping).

### 5.4 Coercion and comparison rules

- **No silent cross-type coercion.** `status == "500"` where the catalog knows
  `status` is int is a **plan-time error**: ``did you mean `status == 500`, or
  `string(status) == "500"`?``
- When the type is unknowable at plan time (freshly parsed dynamic field), a runtime
  comparison between incompatible types yields null and increments a per-query warning
  counter surfaced in `meta.stats`. It is never a hybrid numeric/lexical guess.
- String comparison is lexical: `"10" < "2"` is true. To compare numerically, cast.
- `int op int → int`; `int op float → float`; `/` on two ints is integer division
  when both are ints (`5 / 2 == 2`); use `float(x)` for fractional results. `%` is
  int-only. Division by zero → null. Int overflow wraps per two's complement.
- `+` concatenates only when **both** operands are strings; string + number is a
  plan-time error with a suggested cast.
- timestamp − timestamp → duration; timestamp ± duration → timestamp;
  duration ± duration → duration; duration * / number → duration;
  duration / duration → float.
- Casting: `int(x)`, `float(x)`, `string(x)`, `bool(x)`, `timestamp(x[, layout])`,
  `duration(x)` → null on failure. Strict variants `int!(x)` etc. fail the query with
  row context. `duration()` accepts strings (`"100ms"`); numeric → duration is written
  as multiplication (`n * 1ms`).
- Array index out of bounds → missing. `obj.absent` → missing. `?.` is safe
  navigation on null/missing/non-object.

### 5.5 Case sensitivity (one table)

| Surface | Rule |
|---|---|
| field names | case-sensitive |
| keywords / function names | case-insensitive (lowercase canonical) |
| `==`, `<` etc. on strings | case-sensitive |
| `has` | **always case-insensitive** (the term index is lowercase — honesty) |
| `contains` | case-insensitive by default; `contains_cs` variant |
| `glob` | case-sensitive |
| regex (`matches`, `extract`) | case-sensitive; `(?i)` inline flag |
| bare-term search sugar | case-insensitive |

## 6. Text search semantics

### 6.1 The tokenizer contract (language-level semantics)

The inverted index over `_raw` tokenizes as: **lowercase; tokens are runs of ASCII
alphanumerics and Unicode letters/digits; everything else delimits; no minimum or
maximum token length.** `has` matches exactly these tokens. This contract is part of
the language: changing the tokenizer is a breaking language change.

### 6.2 Cost tiers

| Function | Semantics | Index use | Tier |
|---|---|---|---|
| `has(field, "term")` | whole-token match, case-insensitive; multi-word argument means all tokens present in order-independent conjunction | FST term index + roaring posting lists | fast |
| `contains(field, "sub")` | substring, case-insensitive (`contains_cs` for sensitive) | bloom/term candidates via literal extraction, then verify | moderate |
| `glob(field, "a-*")` | glob match, case-sensitive | literal-prefix extraction when possible | moderate |
| `matches(field, r"…")` | regex (linear-time engine) | literal extraction when the pattern contains required literals; else full scan + lint | slow |

These are scalar predicate **functions** usable anywhere an expression is valid; on
`_raw` at the top of a pipeline the planner pushes them into the scan.

## 7. Parse subsystem (schema-on-read core)

### 7.1 Stage syntax

```
parse <format>[(<args>)] [from <field>] [into (<name> [as <type>], ...)]
      [prefix <p>] [on_error propagate|null|drop|strict]
```

- Default input field is `_raw`. `prefix log.` namespaces extracted fields.
- **Formats**: `json`, `logfmt`, `kv(sep=…, assign=…)`,
  `pattern "<ip> - <user> [<ts>] \"<method> <path>\""` (fast non-regex tier; `<_>`
  skips), `regex r"user=(?<user>\w+)"`, plus the 15 named formats retained from
  `pkg/engine/unpack`: `syslog`, `combined`, `clf`, `nginx_error`, `cef`, `docker`,
  `redis`, `apache_error`, `postgres`, `mysql_slow`, `haproxy`, `leef`, `w3c` (and
  `kv`, `pattern` above).
- **Fallback chains**: `parse first_of(json, logfmt)` — first format that succeeds
  per row wins.
- **Typed captures**: `into (status as int, dur as duration)` — coercion at
  extraction; failure follows `on_error`. `into` also acts as a projection of declared
  captures for formats with dynamic output.
- **Merge rule**: parse stages never delete columns and **never silently overwrite an
  existing non-null field**; on collision the existing value wins and a per-query
  warning counter increments.

### 7.2 `on_error` modes

| Mode | Row | Fields | Error columns |
|---|---|---|---|
| `propagate` (default) | survives | best-effort | `_error`, `_error_detail` populated |
| `null` | survives | null/missing | not populated |
| `drop` | removed | — | — |
| `strict` | query fails with first offending row context | — | — |

Default is `propagate` everywhere, including MV definitions; MV creation emits a lint
suggesting `strict` (§14, D1).

### 7.3 Error columns

- `_error: string` — short machine-stable code+message, e.g.
  `"parse:json: unexpected token at offset 42"`.
- `_error_detail: object` — `{stage: int, format: string, code: string,
  message: string, offset: int?}`. Multiple failing stages append to an array under
  `_error_detail.stages` while `_error` keeps the first failure.

Debugging workflow (first-class):
`| parse json | where exists(_error) | keep _error, _error_detail, _raw`.

### 7.4 `describe`

`| describe` replaces `glimpse`/`fieldsummary`: emits one row per field of the current
stream with `field`, `type`, `coverage` (fraction non-missing), `distinct_est`, and
`top_values` (array of `{value, count}`).

### 7.5 Ingest-time relationship

Ingest-time top-level extraction is retained as an acceleration with identical
semantics: a stored column is indistinguishable from a query-time-extracted field.
v1 ingest changes (Phase 5+): record `_time_source: parsed|header|ingest_clock`
provenance instead of a silent `time.Now()` fallback; store ES-bulk/OTLP nested values
consistently. Field promotion (`lynxdb fields promote user.id`) is a follow-on, not v1.

## 8. Core operator catalog

Core operators (18). "Streaming": row = row-at-a-time safe for live tail;
acc = accumulating/blocking.

| Operator | Syntax | Streaming | Semantics |
|---|---|---|---|
| `from` | `from <sources>[range] [sugar-terms]` | row | scan stage; only valid first (or as CTE reference target) |
| `where` | `where <expr>` | row | typed predicate filter |
| `parse` | §7 | row | schema-on-read extraction |
| `extend` | `extend x = <expr>, y = <expr>` | row | add/replace computed columns, left-to-right |
| `keep` | `keep f1, f2, pat*` / `keep * except f1, pat*` | row | projection, order-preserving |
| `drop` | `drop f1, pat*` | row | remove columns |
| `rename` | `rename old as new, …` | row | rename columns |
| `stats` | `stats <agg> [as name], … [by k1, k2, bin(_time, 5m)]` | acc | grouped aggregation; binned key emits as `_time`; output schema explicit |
| `eventstats` | `eventstats <agg> [as name], … [by keys]` | acc | aggregates appended to every row |
| `streamstats` | `streamstats [window=N] [current=true\|false] <agg\|window-fn> [as name], … [by keys]` | row | running/windowed values in row order |
| `sort` | `sort -f1, +f2, f3` (`-` desc, default asc) | acc | external merge sort, spill-capable |
| `head` | `head N` | row | first N rows (with TopK pushdown after sort) |
| `tail` | `tail N` | acc | last N rows |
| `dedup` | `dedup [N] f1, f2` | row | keep first N (default 1) rows per key |
| `join` | `join [type=inner\|left\|outer] on f1[, f2] with ($cte \| [<pipeline>])` | acc | hash join; default `type=inner` (plain inner, never innerunique) |
| `union` | `union ($cte \| [<pipeline>]) [, …]` | row | append rows; schemas merge by name with null-padding |
| `explode` | `explode arr [as elem]` | row | one row per array element; missing/empty array drops the row |
| `describe` | `describe` | acc | stream schema/coverage summary (§7.4) |

Management stages (not relation transforms, registry class `management`):
`materialize "<name>" [retention=<dur>]` (terminal stage, creates an MV),
`tee "<sink>"` (side-effect passthrough), `use <fragment>` (parse-time fragment
expansion, preserved from RFC-001).

## 9. Sugar tier and investigation helpers

Every sugar form has exactly one mechanical expansion recorded as
`Rewrite{Before, After, Reason}` and shown by `--show-rewritten` / `meta.rewrites` /
EXPLAIN. Anything without a mechanical desugar is rejected from this tier.

### 9.1 Sugar (pure desugar)

| Sugar | Expansion |
|---|---|
| search-position terms (§3.1) | `where …` conjunction |
| `top [N] field` | `stats count() as count by field \| sort -count \| head N` (N default 10) |
| `rare [N] field` | `stats count() as count by field \| sort +count \| head N` |
| `every <dur> [by k…] stats <aggs>` | `stats <aggs> by [k…,] bin(_time, <dur>)` |
| `rate [per <dur>] [by k…]` | `every <dur(def 1m)> [by k…] stats count() as rate` |
| `latency f [every <dur>] [by k…]` | `stats p50(f), p95(f), p99(f), count() [by k…, bin(_time, <dur>)]` |
| `percentiles f [by k…]` | `stats p50(f) as p50_f, p75(f) as p75_f, p90(f) as p90_f, p95(f) as p95_f, p99(f) as p99_f [by k…]` |
| `proportion <pred> as <name> [every <dur>] [by k…]` | `stats count(where <pred>) as <name>_num, count() as <name>_den [by …] \| extend <name> = <name>_num / <name>_den` |
| `facets f1, f2 [limit=N]` | per-field `stats count() by f` fan-out unioned with `_facet`/`_value`/`count` columns (RFC-001 §5.8 shape, via `union`) |
| `impact [agg] by k…` | `stats <agg> as v by k… \| eventstats sum(v) as total_v \| extend pct_v = v / total_v \| sort -pct_v` |
| `baseline f window=N [by k…]` | `streamstats current=false window=N avg(f) as baseline_f, stdev(f) as stdev_f [by k…] \| extend delta_f = f - baseline_f, z_f = if(stdev_f > 0, delta_f / stdev_f, null)` |
| `changes f [by k…]` | `sort +_time \| streamstats current=false last(f) as previous_f [by k…] \| where exists(previous_f) and f != previous_f` |
| `exemplars [N] [by k…]` | `sort -_time \| dedup N k…` (global: `head N`) |

### 9.2 Runtime helpers (registry class `helper`; declared cost, stable semantics)

`patterns [field=_raw] [max_templates=N]`, `compare [previous] <dur>` (window replay
of the pipeline prefix; emits `previous_*`/`change_*` columns), `outliers field=f
[method=iqr|zscore|mad] [threshold=N]`, `sessionize [maxpause=<dur>] [by k…]`,
`transaction [maxspan=<dur>] [startswith=<expr>] [endswith=<expr>] k…`,
`trace [trace_id=f] [span_id=f] [parent_id=f]`, `topology [source_field=f]
[dest_field=f] [weight_field=f]`, `correlate f1 f2 [method=pearson|spearman]`,
`rollup <dur>[, <dur>…] [by k…]`, `xyseries x y value`.

`patterns` and `compare` are logical-IR nodes (they need runtime support); the rest
map 1:1 to preserved pipeline operators.

## 10. Function catalog

Every function declares a **fallibility class**: *infallible*, *null-on-failure*
(default), or *strict-variant-available* (`!` suffix). The registry
(`pkg/lynxflow/registry`) is the single source of truth; this section is the frozen
v1 surface.

- **Conversion**: `int`, `float`, `string`, `bool`, `timestamp(x[, layout])`,
  `duration` (+ `!` variants).
- **Conditional/null**: `if(cond, then, else)`,
  `case(cond1, v1, cond2, v2, …[, default])`, `coalesce(…)` / `??`, `nullif(a, b)`,
  `exists`, `is_null`, `is_missing`, `typeof`.
- **String**: `len`, `lower`, `upper`, `trim/ltrim/rtrim(s[, chars])`,
  `substr(s, start[, len])` (0-based), `replace(s, r"re", with)`, `split(s, sep) →
  array`, `join(arr, sep)`, `starts_with`, `ends_with`, `contains`, `contains_cs`,
  `glob`, `printf(fmt, …)`, `urldecode`, `url_parse → object`, `path_normalize`,
  `useragent_parse → object` (optional build).
- **Text search**: `has` (§6).
- **Regex**: `matches(s, r"…")`, `extract(s, r"…")` (first capture group),
  `extract_all(s, r"…") → array`.
- **Math**: `abs`, `round(x[, digits])`, `floor`, `ceil`, `sqrt`, `ln`,
  `log(x[, base])`, `exp`, `pow`, `clamp(x, lo, hi)`, `bucket(x, [bounds])`; trig
  retained (`sin`, `cos`, `tan`, `asin`, `acos`, `atan`, `atan2`).
- **Time**: `now()`, `bin(ts, dur)`, `strftime(ts, fmt)`, `strptime(s, fmt)`
  (null-on-failure; distinguishable from null input via `_error`-free semantics:
  use `strptime!` to assert), `time_of_day(ts) → duration`, `day_of_week(ts) → int`.
- **Hash/network**: `md5`, `sha1`, `sha256`, `xxhash64`, `cidr_match("net/mask", ip)`,
  `ip_parse → object`, `ipmask(mask, ip)`.
- **Array**: `len`, `arr[i]` (0-based; negative from end), `slice(arr, start[, end])`,
  `array_concat`, `array_distinct`, `array_sort`, `flatten`, `any(arr, x -> p)`,
  `all(arr, x -> p)`, `filter(arr, x -> p)`, `map(arr, x -> e)`, `x in arr`.
- **Object**: `obj.k`, `obj["k"]`, `keys(obj) → array`, `values(obj) → array`,
  `merge(a, b)`, `has_key(obj, k)`, `to_json(x) → string`, `from_json(s) → any`.
- **Aggregates** (in `stats`/`eventstats`/`streamstats`): `count([where p])`,
  `count(x[, where p])`, `sum(x[, where p])`, `avg`, `min`, `max`, `dc`, `estdc`,
  `perc(x, p)` + alias family `p50/p75/p90/p95/p99` **only** (the
  `exactperc95`/`upperperc95`/`percentile95` zoo is killed), `stdev`, `var`, `mode`,
  `first`, `last`, `earliest`, `latest`, `values(x) → array` (distinct),
  `list(x) → array`, `rate`, `per_second`.
- **Window** (streamstats context): `lag(x[, n])`, `lead(x[, n])`, `row_number()`,
  `running_sum(x)`, `moving_avg(x, n)`.

## 11. Error reporting and diagnostics

- **Parse errors**: span + caret rendering in CLI/TUI; `expected` token set;
  did-you-mean for misspelled operators; recovery at `|` boundaries yields all errors
  in one pass. Structured JSON `{code, message, span: {start, end}, expected[],
  suggestion}` — additive extension of today's contract.
- **Semantic errors** (plan-time): unknown field (catalog did-you-mean), type mismatch
  with fix-it casts, aggregate misuse, group-by-missing-field, streaming-unsafe stage
  in live tail.
- **Runtime warnings**: per-query counters (failed casts, incompatible comparisons,
  parse failures by stage) in `meta.stats`; never silent.
- **Lints**: registry-based; new catalog seeded from the proven RFC-001 set (leading
  wildcard, broad scope, null-group share, regex-without-literal-anchor,
  sugar-available, old-SPL2-spelling did-you-mean). Same REST/CLI/TUI plumbing
  (`meta.lints`).
- **Rewrite transparency**: retained verbatim (`meta.rewrites`, `--show-rewritten`).
- **EXPLAIN**: renders the logical IR with per-node index interaction (term-index
  pushdown, bloom/RG skip estimates, BSI usage, spill capability); EXPLAIN ANALYZE
  adds per-node rows/time/spill events. `meta.explain` block kept.
- Error-message text is a **tested artifact**: golden error-message files (Phase 3).

## 12. Registry: one source of truth

`pkg/lynxflow/registry` declares, as data:

- **Operators**: name, class (`source|core|sugar|helper|management`), streaming class,
  positional shape, option schema (name/type/required/default), desugar reference,
  doc string, examples.
- **Functions**: name, category, arity, argument/result types, fallibility class,
  strict-variant flag, doc string.
- **Aggregates**: name, arity, `where`-clause support, result type, doc string.

Consumers: the parser (stage/option parsing tables), the desugarer, the semantic
analyzer (typing), the VM compiler (Phase 7 function registry keys off the same
names), autocomplete catalogs (shell Go + web via `/api/v1/catalog`), EBNF generation,
docs stubs, and the LLM cookbook. Hand-maintained duplicates of any of these are
deleted in Phase 10.

## 13. Example queries (before → after)

The 18 canonical examples; the acceptance corpus
(`pkg/lynxflow/testdata/corpus/corpus.jsonl`) extends these to 50.

1. `search timeout error` → `from app[-1h] error timeout`
   *(desugars: `| where has(_raw, "error") and has(_raw, "timeout")`)*
2. `index=nginx status>=500 "timeout"` → `from nginx[-1h] timeout status>=500`
   *(desugars: `| where has(_raw, "timeout") and status >= 500`)*
3. `| unpack_json from _raw | where status>=500` → `| parse json | where status >= 500`
4. `| unpack_logfmt prefix log.` → `| parse logfmt prefix log.`
5. `| rex field=_raw "user=(?P<user>\w+)"` →
   `| parse regex r"user=(?<user>\w+) ip=(?<ip>[\d.]+)" into (ip as string)`
6. `| table host, status` → `| keep host, status`; `| fields - _raw` → `| drop _raw`
   or `| keep * except _raw`
7. `| rename duration_ms AS latency` → `| rename duration_ms as latency`
8. `| eval is_err=if(status>=500,1,0)` → `| extend is_err = status >= 500` *(bool)*
9. `| stats count, avg(dur) by service` → `| stats count(), avg(dur) by service`
10. `| timechart count span=5m by service` →
    `| stats count() by service, bin(_time, 5m)` *(sugar: `| every 5m by service stats count()`)*
11. `| stats count as total, count(eval(status>=500)) as errors | eval rate=errors/total` →
    `| stats count(where status >= 500) as errors, count() as total by service | extend error_rate = errors / total`
    *(sugar: `| proportion status >= 500 as error_rate by service`)*
12. `| stats perc50(d), perc95(d), perc99(d) by endpoint` →
    `| stats p50(dur), p95(dur), p99(dur) by endpoint, bin(_time, 5m)`
    *(sugar: `| latency dur every 5m by endpoint`)*
13. nested JSON *(impossible today)* →
    `| parse json | where user.role == "admin" and any(tags, t -> t.name == "vip")`
14. `| where isnotnull(amount)` → `| where exists(amount)`;
    defaulting: `| extend amount = amount ?? 0`
15. parse-failure debugging *(impossible today)* →
    `| parse json | where exists(_error) | keep _error, _error_detail, _raw | head 20`;
    coverage: `| parse json | describe`
16. `lynxdb explain 'from nginx[-1h] timeout | stats count() by host'` → IR plan with
    `Scan(nginx, -1h, term:"timeout" via FST, est. RG skip 97%)`
17. `→ from nginx | parse combined | where status == 500 and has(_raw, "upstream")`
18. `→ from app[-1h] | parse first_of(json, logfmt) | keep _time, service, status, dur | sort -dur | head 50`

## 14. Decision log

Resolutions of PLAN.md §23 open questions and of conflicts between PLAN.md and the
synthesis spec. PLAN.md governs where it speaks; these are the calls where it did not.

| # | Decision | Rationale |
|---|---|---|
| D1 | Default `on_error` is `propagate` everywhere, including MV definitions; MV creation emits a lint suggesting `strict`. | One consistent default beats context-dependent defaults (PLAN §23.1 leaning). |
| D2 | Ingest-time nested extraction and field promotion deferred to a follow-on RFC. | PLAN §23.2. |
| D3 | Cluster IR serialization out of scope; the IR is designed serializable. | PLAN §23.3. |
| D4 | `like` is **dropped**. A lint maps it to `contains`/`matches`/`glob`. | Three tiers already cover it (PLAN §23.4 leaning). |
| D5 | Lambda syntax locked as `x -> expr`. | PLAN §23.5. |
| D6 | Language name stays **LynxFlow**. | PLAN §23.6; zero technical impact. |
| D7 | Error columns are `_error` / `_error_detail` (single underscore). | PLAN §10.2 wording wins over synthesis `__error__`. |
| D8 | `==` compares, `=` binds. | PLAN §9.2 wins over synthesis `=` equality. |
| D9 | Text tiers are `has`/`contains`/`matches` (+`glob`); synthesis's `term()` is not a surface name. | PLAN §8/§17 wins. |
| D10 | `has`/`contains`/`matches`/`glob` are **functions only**, not infix operators. PLAN §9.2 listed them as comparison operators, but every PLAN example uses call form; one spelling wins. | Predictability; one way to do things. |
| D11 | Null and missing **each form their own group key** (with a high-share lint). | PLAN §10.2 wins over synthesis (which merged them). |
| D12 | `contains` is case-insensitive by default with `contains_cs` variant. | PLAN §10.3 wins over synthesis (case-sensitive default). |
| D13 | Aggregation stage is named `stats` (not synthesis's `aggregate(...)`); computed columns are `extend` (not `derive`); projection is `keep`/`drop` (not `select`/`exclude`). | PLAN §17 examples win. |
| D14 | Core operator count is 18 (§8): the 16 obvious stages plus `eventstats` and `streamstats` as named core operators (the IR `Window` node covers both). | PLAN §13/§15 require window support; Splunk-familiar names maximize migration predictability. |
| D15 | No `search` stage and no `filter` stage. Search sugar lives only in the `from` stage; mid-pipeline text search is `where has(...)`. | One grammar, one filter stage. |
| D16 | Quoted phrases in search sugar desugar to `contains(_raw, phrase)` (CI); bare words to `has`. | Honest about what the term index can and cannot do. |
| D17 | Glob values in key:value sugar desugar to `glob(field, pattern)`, case-sensitive. Field-value comparisons stay CS; only raw-text term search is CI. | Single clean CS/CI rule (§5.5). |
| D18 | Single-quoted strings/identifiers are a parse error with fix-it. Identifier quoting is backticks only; double quotes are always strings. | Kills the RFC-001 L012 ambiguity class outright. |
| D19 | F-strings are removed in v1; `printf` and string `+` cover the use cases. | Not in PLAN §9.3's literal table; may return by amendment. |
| D20 | Duration units: `ns,us,ms,s,m,h,d,w`; no month/year. `duration()` casts strings only; numeric→duration via `n * 1ms`. | Months/years are calendar-dependent; explicit is better. |
| D21 | CTEs require `let`: `let $x = <pipeline>;`. `| let` as an eval alias is killed (use `extend`). | `let` had two meanings in RFC-001 practice; keep one. |
| D22 | Sort is prefix-form only: `sort -f, +f, f`. `sort by f desc` is a parse error with fix-it. | RFC-001 canon already preferred prefix form (L022). |
| D23 | `dedup [N] f1, f2` keeps RFC-001's shape (no `by`). | Muscle memory; no ambiguity to resolve. |
| D24 | Comparisons are non-associative (`a<b<c` is a parse error with fix-it). | Silent boolean-chaining bugs. |
| D25 | Dotted-path resolution: flat column first, then object access; backticks force flat (§4.4). The implicit `_raw` JSON fallback is removed. | Mirrors existing data while making schema-on-read explicit. |
| D26 | Integer `/` is integer division; mixed int/float promotes to float. | Matches Go/SQL expectations; explicit `float()` for ratios. |
| D27 | `top`/`rare` keep no `by` variant; per-group top-N is the documented `streamstats row_number()` idiom until a dedicated form ships. | The `by` variant has no mechanical desugar without window functions. |
| D28 | `count` without parentheses is a parse error with fix-it `count()`. | SPL1-ism; mandatory parens (Splunk's own SPL2-2026 agrees). |

## 15. RFC-001 capability disposition

Every RFC-001 capability, mapped or killed. Verdicts: **KEEP** (same surface),
**MAP** (new spelling, same capability), **KILL** (capability or spelling removed;
migration note), **DEFER** (explicitly post-v1).

### 15.1 Language structure

| RFC-001 capability | Verdict | v2 form / note |
|---|---|---|
| Three expression contexts (SEARCH/WHERE/EVAL) | KILL | one grammar (§4) |
| SEARCH precedence (OR before AND) | KILL | standard precedence everywhere |
| `XOR` | KILL | `a != b` on bools |
| CTEs `$x = …;` | MAP | `let $x = …;` (D21) |
| Free-hand search + default scope rewrite | KEEP | §3 |
| Implicit `FROM main` for bare pipelines | KEEP | implicit `from <default>` + rewrite |
| `FROM`/`INDEX` source stage | MAP | `from` only; `index` keyword killed |
| `index=x` / `source=x` / `sourcetype=y` prefix compat | KILL | `from x`; `where _source == "x"`; one release of did-you-mean hints |
| `earliest=`/`latest=`/`_index_earliest=`/`_index_latest=` | KILL | bracket ranges `[a..b]`; `where _indextime …` |
| Source globs/excludes/`*`/CTE refs | KEEP | §2.5 |
| Bracket time ranges + snap `[@d]` | KEEP | §2.4 |
| `--` comments | KILL | `//` and `/* */` |
| F-strings | KILL | `printf`, string `+` (D19) |
| Quoted identifiers `'x'`/`"x"` | MAP | backticks (D18) |
| Glob literals in identifiers | KEEP | sugar position + `glob()` |
| Regex as quoted string + `=~`/`!~` | MAP | `matches(f, r"…")`; raw strings |
| Linear-regex default, PCRE2 opt-in | KEEP | §2.3 |
| `LIKE` | KILL | `contains`/`glob`/`matches` (D4) |
| `BETWEEN`/`IN`/`IS NULL` | MAP | `between…and…`, `in [..]`, `is_null()`/`exists()` |
| `?.` and `??` | KEEP | §4 |
